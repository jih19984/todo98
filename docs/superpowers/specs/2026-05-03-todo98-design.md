# Todo98 기획 문서

## 한 줄 정의

Todo98은 **윈도우 98 / Y2K 감성의 개인용 TODO 웹앱**입니다.

사용자는 Google 또는 Kakao로 로그인하고, 레트로 OS 창 안에서 오늘 할 일과 전체 할 일을 관리합니다. 운영 비용은 최대한 무료로 유지하고, 사용자가 돈을 쓰는 부분은 **커스텀 도메인 구매**만으로 제한합니다.

## 목표

- 매일 개인 할 일을 빠르게 정리할 수 있는 TODO 앱을 만든다.
- 첫 화면에서 Retro OS Window / Y2K 감성이 바로 느껴지게 한다.
- Google 로그인과 Kakao 로그인을 실제 운영 수준으로 연결한다.
- Supabase Auth와 Supabase DB를 사용해 사용자별 할 일을 안전하게 저장한다.
- Vercel 무료 플랜으로 배포한다.
- 구매한 커스텀 도메인을 Vercel에 연결한다.
- Apple 로그인은 이번 MVP에서 제외한다.

## 제품 범위

### 공개 랜딩 페이지

방문자가 처음 보는 페이지는 작은 레트로 데스크톱처럼 구성합니다.

예시 창 이름:

- `Todo98`
- `Login.exe`
- `Today.tasks`

랜딩 페이지에서 보여줄 핵심 메시지는 다음과 같습니다.

- Todo98은 개인 할 일을 관리하는 TODO 앱이다.
- Google 또는 Kakao 계정으로 시작할 수 있다.
- 레트로 감성은 강하지만 실제 사용성은 현대적으로 만든다.

주요 버튼:

- Google로 계속하기
- Kakao로 계속하기

### 로그인

인증은 Supabase Auth를 사용합니다.

지원 로그인:

- Google OAuth
- Kakao OAuth

로그인 성공 후에는 바로 TODO 앱 화면으로 이동합니다.

로컬 개발 또는 배포 환경에서 Supabase 환경변수가 빠져 있으면, 조용히 실패하지 않고 설정이 필요하다는 안내를 보여줍니다.

### TODO 앱 화면

로그인 후 사용자는 하나의 메인 창에서 할 일을 관리합니다.

기본 보기:

- 오늘
- 전체
- 완료

사용자가 할 수 있는 일:

- 할 일 추가
- 할 일 수정
- 할 일 삭제
- 완료 처리
- 완료 취소
- 제목 입력
- 메모 입력
- 마감일 설정
- 우선순위 설정: 낮음, 보통, 높음

이번 MVP에서는 기능을 과하게 늘리지 않습니다. 태그, 드래그 정렬, 반복 일정, 통계는 배포까지 끝난 뒤 시간이 남을 때 추가 후보로 둡니다.

### 계정 / 로그아웃

작은 계정 창 또는 메뉴에서 다음 정보를 보여줍니다.

- 로그인한 이메일
- 로그인 제공자
- 로그아웃 버튼

## UI 방향

선택된 스타일은 **Retro OS Window**입니다.

시각 규칙:

- 윈도우 98 / 2000 초반 느낌의 창 UI
- 타이틀바, 픽셀 느낌의 테두리, 작은 OS 버튼
- `Today.tasks`, `NewTask.exe`, `Settings.exe` 같은 파일명 감성
- 전기 파랑, 민트, 레몬, 핑크 같은 Y2K 포인트 컬러
- 기본 화면은 너무 어둡지 않게 유지
- 버튼은 현대적인 둥근 pill 스타일보다 사각형 또는 약한 라운드 중심
- 모바일에서는 창이 세로로 쌓이고 터치하기 쉬워야 함

중요한 원칙:

레트로 감성은 강하게 주되, 사용성이 희생되면 안 됩니다. 할 일 추가, 완료, 수정은 3초 안에 이해되어야 합니다.

## 데이터 구조

### `tasks` 테이블

필드:

- `id`: UUID 기본키
- `user_id`: Supabase auth user id
- `title`: 할 일 제목, 필수
- `note`: 메모, 선택
- `due_date`: 마감일, 선택
- `priority`: 우선순위, `low`, `normal`, `high`
- `completed_at`: 완료 시간, 선택
- `created_at`: 생성 시간
- `updated_at`: 수정 시간

인덱스:

- `user_id`
- `user_id, due_date`
- `user_id, completed_at`

보안:

- Supabase Row Level Security를 켭니다.
- 사용자는 자기 `user_id`에 해당하는 할 일만 조회, 추가, 수정, 삭제할 수 있습니다.

## 기술 구조

### 프론트엔드

사용 기술:

- Next.js
- TypeScript
- Supabase JS SDK

주요 구성:

- Supabase 클라이언트 설정
- 랜딩 / 로그인 페이지
- OAuth 콜백 처리
- 로그인된 사용자의 TODO 앱 페이지
- 할 일 CRUD 서비스
- Retro OS 컴포넌트: 창, 타이틀바, 버튼, 입력창, 모달

### 백엔드

별도 서버는 만들지 않습니다.

Supabase에서 담당:

- Google 로그인
- Kakao 로그인
- Postgres DB
- Row Level Security

## 환경변수

앱에 필요한 환경변수:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

Supabase 대시보드에서 필요한 설정:

- Google OAuth Client ID
- Google OAuth Client Secret
- Kakao REST API Key
- Kakao Client Secret, 사용하는 경우

Vercel에도 같은 Supabase 환경변수를 등록합니다.

## 배포와 도메인

배포:

- Vercel Hobby 무료 플랜

도메인:

- 사용자가 원하는 도메인을 구매
- Vercel 프로젝트에 도메인 연결
- Vercel이 안내하는 DNS 레코드 설정

운영 배포 순서:

1. Vercel에 프로젝트 배포
2. 커스텀 도메인 구매
3. Vercel 프로젝트에 도메인 추가
4. DNS 레코드 설정
5. Supabase Auth redirect URL에 운영 도메인 추가
6. Google 개발자 콘솔에 운영 redirect URL 추가
7. Kakao 개발자 콘솔에 운영 redirect URL 추가
8. 운영 도메인에서 Google/Kakao 로그인 검증

## 에러 처리

로그인:

- OAuth 실패 시 안내 메시지 표시
- Supabase 환경변수가 없으면 설정 필요 메시지 표시

할 일:

- 제목이 비어 있으면 저장 불가
- 저장, 수정, 삭제 실패 시 작은 레트로 경고창 표시
- 애니메이션보다 데이터 정확성을 우선

## 테스트 기준

구현은 TDD 방식으로 진행합니다.

자동 테스트:

- 오늘 / 전체 / 완료 필터
- 빈 제목 저장 방지
- 할 일 완료 / 완료 취소
- 할 일 수정
- 할 일 삭제
- Google 로그인 버튼이 Supabase Google OAuth를 호출하는지
- Kakao 로그인 버튼이 Supabase Kakao OAuth를 호출하는지
- 로그인하지 않은 사용자가 앱 페이지에 접근할 수 없는지

수동 검증:

- 데스크톱 화면
- 모바일 화면
- 운영 도메인 HTTPS 접속
- 운영 도메인 Google 로그인
- 운영 도메인 Kakao 로그인
- Vercel 커스텀 도메인 연결 상태
